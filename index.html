java IO

Veröffentlicht am 2016-09-28   |  

java程序中,数据的输入/输出一‘流’的方式进行。J2SDK提供了各种‘流’类，用以获取不同种类的数据；程序中通过标准的方法输入或输出数据。

JAVA.IO

按方向可以分为输入流和输出流。
按数据处理单位可以分为字节流和字符流
按照功能不同可以分为节点流和处理流。
| 类型 | 字节流 | 字符流 |
| ————- |:————-:| :————-:|
|输入流 | IntputStream | Reader |
| 输出流 | OutputStream | Writer |

InputStream类

向程序中输入数据，且数据的单位为字节（8bit）,即从文件中读内容。
子类：

  • FileInpuStream(节点流)
  • PipedInputStream(节点流)
  • FilterInputStream（处理流）
    • LineNumberInputStream（处理流）
    • DateInputStream（处理流）//（数据流）
    • BufferedInputStream（处理流）//（缓冲流）
    • PushbackInputStream（处理流）
  • ByteArrayInputStream(节点流)
  • SequenceInputStream（处理流）
  • StringBufferInputStream(节点流)
  • ObjectInputStream（处理流）

方法

int read()//读取一个字节并以整数形式返回。
int read(byte[] buffer)//读取一系列字节并存储到一个数组buffer，返回实际读取的字节数
void close()//关闭流释放内存

OutputStream类

从程序中输出数据，且数据的单位为字节（8bit）；往文件里写数据。

  • FileOutpuStream(节点流)
  • PipedOutputStream(节点流)
  • FilterOutputStream（处理流）
    • DateOutputStream（处理流）
    • BufferedOutputStream（处理流）
    • PrintStream（处理流）//（打印流）
  • ByteArrayOutputStream(节点流)
  • ObjectOutputStream（处理流）

方法

void write(int b)//向输出流中写入一个字节数据，字节数据为参数b的低8位
void write(byte[] b)//将一个字节类型的数组中的数据写入输出流
void write(byte[] b,int off, int len)//将一个字节类型的数组中的从指定位置（off）开始的len个字节写入到输出流。
void close()//关闭流释放内存资源
void flush()//将输出流中缓冲的数据全部写出目的地

Reader

继承自Reader的流都是用于向程序中输入数据，且数据的单位为字符（16bit）；

  • BufferedReader（处理流）//（readline方法）
    • LineNumberInputStream（处理流）
  • CharArrayReader（节点流）
  • IntputStreamReader(处理 流)//（转换流）
    • FileReader(节点流)
  • PipedReader(节点流)
  • FilterInputStream（处理流）
    • PushbackInputStream（处理流）
  • StringReader(节点流)

Writer

继承自Reader的流都是用 于从程序中输出数据，且数据的单位为字符（16bit）；

  • BufferedWriter（处理流）
  • CharArrayWriter（节点流）
  • OutputStreamWriter(处理流)//（转换流）
    • FileWriter(节点流)
  • PipedWriter(节点流)
  • PrintWriter（处理流）
  • StringWriter(节点流)

转换流

Collection接口

Collection接口定义了存取一组对象的方法

方法：

  • int size();
  • boolean isEmpty();
  • void clear();
  • boolean contains(Object element);
  • boolean add(Object element);//多态思想
  • boolean remove(Object element);
  • Iterator iterator();
  • boolean containsAll(Collection c);
  • boolean addAll(Collection c);
  • boolean removeAll(Collection c);
  • boolean retainAll(Collection c);
  • Object[] toArray();

重写存储对象的equals方法必须重写其hashCode()方法；
Object的equals方法即对内存中指向相同；

Set接口

Collection的子接口，定义了存储方式；
Set中的数据对象没有顺序且不可重复。

List中的数据对象有顺序且可以重复。

  • Onject set(int index, Object element);
  • void add(int index, Object element);
  • Object remove(int index);
  • int indexOf(Object o);`
  • int lastIndexOf(Object o);

Map接口

定义了存储“键-值对”的方法；
实现类有HashMap和TreeMap等；
Map类中存储的键-值对通过键来标识，所以键值不能重复。（这里比较的是hashcode）

  • Object put (Object key, Object value);//若key不存在则返回null；
  • Object get (Object key);
  • Object remove (Object key);
  • bollean containsKey (Object key);
  • boolean containsValue (Object value);
  • int size();
  • boolean isEmpty();
  • void putAll(Map t);
  • void clear();

Iterator接口

所有实现了Collection接口的容器类都有一个iterator方法用以返回一个实现了Iterator接口的对象。
Iterator对象称作迭代器，用以方便的实现对容器内元素的遍历操作；

方法：

  • boolean hasNext();
  • Object next();
  • void remove();

java.util.Collections类

java.util.Collections类提供了一些静态方法实现基于List容器的一些常用算法。

  • void sort(List)//排序
  • void shuffle(List)//随机排序
  • void reverse(List)//逆序排序
  • void fill(List, Object)//用一个特定对象重写整个容器
  • void copy(List dest, List src)//将src List容器拷贝到dest List容器
  • int binarySearch(List, object)//折半查找

Comparable

Collections中排序的依据
public int compareTo(Object obj)

  • 返回0表示 this == obj
  • 返回正数表示 this > obj
  • 返回复数表示 this < obj
