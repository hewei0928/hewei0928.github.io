java IO

Veröffentlicht am 2016-09-28   |  

java程序中,数据的输入/输出一‘流’的方式进行。J2SDK提供了各种‘流’类，用以获取不同种类的数据；程序中通过标准的方法输入或输出数据。

JAVA.IO

按方向可以分为输入流和输出流。
按数据处理单位可以分为字节流和字符流
按照功能不同可以分为节点流和处理流。
| 类型 | 字节流 | 字符流 |
| ————- |:————-:| :————-:|
|输入流 | IntputStream | Reader |
| 输出流 | OutputStream | Writer |

InputStream类

向程序中输入数据，且数据的单位为字节（8bit）,即从文件中读内容。
子类：

  • FileInpuStream(节点流)
  • PipedInputStream(节点流)
  • FilterInputStream（处理流）
    • LineNumberInputStream（处理流）
    • DateInputStream（处理流）//（数据流）
    • BufferedInputStream（处理流）//（缓冲流）
    • PushbackInputStream（处理流）
  • ByteArrayInputStream(节点流)
  • SequenceInputStream（处理流）
  • StringBufferInputStream(节点流)
  • ObjectInputStream（处理流）

方法

int read()//读取一个字节并以整数形式返回。
int read(byte[] buffer)//读取一系列字节并存储到一个数组buffer，返回实际读取的字节数
void close()//关闭流释放内存

OutputStream类

从程序中输出数据，且数据的单位为字节（8bit）；往文件里写数据。

  • FileOutpuStream(节点流)
  • PipedOutputStream(节点流)
  • FilterOutputStream（处理流）
    • DateOutputStream（处理流）
    • BufferedOutputStream（处理流）
    • PrintStream（处理流）//（打印流）
  • ByteArrayOutputStream(节点流)
  • ObjectOutputStream（处理流）

方法

void write(int b)//向输出流中写入一个字节数据，字节数据为参数b的低8位
void write(byte[] b)//将一个字节类型的数组中的数据写入输出流
void write(byte[] b,int off, int len)//将一个字节类型的数组中的从指定位置（off）开始的len个字节写入到输出流。
void close()//关闭流释放内存资源
void flush()//将输出流中缓冲的数据全部写出目的地

Reader

继承自Reader的流都是用于向程序中输入数据，且数据的单位为字符（16bit）；

  • BufferedReader（处理流）//（readline方法）
    • LineNumberInputStream（处理流）
  • CharArrayReader（节点流）
  • IntputStreamReader(处理 流)//（转换流）
    • FileReader(节点流)
  • PipedReader(节点流)
  • FilterInputStream（处理流）
    • PushbackInputStream（处理流）
  • StringReader(节点流)

Writer

继承自Reader的流都是用 于从程序中输出数据，且数据的单位为字符（16bit）；

  • BufferedWriter（处理流）
  • CharArrayWriter（节点流）
  • OutputStreamWriter(处理流)//（转换流）
    • FileWriter(节点流)
  • PipedWriter(节点流)
  • PrintWriter（处理流）
  • StringWriter(节点流)

转换流

字节流转换为字符流

浮点类型

float 4字节 精度7位。
double 8字节 精度15位。

类型转换

boolean不能与其他类型转换。
容量小的自动转换为容量大的数据。
指数字大而不是占内存字节数
byte,short,char->int->long->float->double;
byte,short,char做运算时自动转换为int
byte: 141->-114;
(000000000 000000000 000000000 10001101)
->01110010;->-114->-1=-115;

final

不允许修改的变量；

基础类型占一个内存

引用类型占两个内存

引用类型全为指针；对象是new出来的，放在堆内存。一小块内存指向一大块。

构造方法没有返回值

方法执行完毕，栈内存中的局部变量销毁

static声明的成员变量：
可通过类名直接访问， 静态方法不针对某个对象调用，所以不能访问非静态成员

子类的构造过程中必须调用父类构造方法

子类中使用super(argument_list)调用父类构造方法；
只用this(argument_list)调用本类的构造方法；
如果调用了super，必须写在子类构造方法内的第一行。
如果子类的构造方法没有显示调用父类构造方法，则系统默认调用父类的无参构造方法，而若父类中没有无参构造方法，则编译出错。

new 出来的对象永远不“==”
.equals()重写后可以用于比较；
new String() equals 相等是因为重写了该方法；

没有import即可使用的类一定在java.lang中

hash 编码用于存储对象在堆内存中的地址

对象转型：
一个基类的引用不可以访问其子类对象新增的成员；
可以使用变量 instanceof 类名来判断引用类型变量所指向的对象是否属于该类或是该类的子类，子类的对象可以作为基类的对象来使用 称作向上转型，反之称为向下转型

多态：
在执行期间 判断所引用的实际类型，根据其实际的类型（即实际new 的class类型，或）调用相应的方法。（方法重写时）。

存在的三个条件

  1. 要有继承。
  2. 要有方法重写。
  3. 父类引用类型指向子类对象。

抽象方法 ： 用于多态的方法。 abstract
抽象类不能实例化对象， 抽象方法只需声明， 不需实现。
抽象方法只用于被重写。

final 的变量：不能被改变；
final 的方法：不能被重写；
final 的类：不能被继承。

接口

一种特殊的抽象类。只包含常量和方法的定义，没有变量和方法的实现。

  1. 接口可以多重实现
  2. 接口中的成员变量属性为public static final；
  3. 接口中只能定义抽象方法，且方法只能为public；
  4. 接口可以继承其他接口，并添加新属性和抽象方法。

}

class Student implements Signer{
}
