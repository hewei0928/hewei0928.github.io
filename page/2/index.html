Iterator接口

所有实现了Collection接口的容器类都有一个iterator方法用以返回一个实现了Iterator接口的对象。
Iterator对象称作迭代器，用以方便的实现对容器内元素的遍历操作；

方法：

  • boolean hasNext();
  • Object next();
  • void remove();

java.util.Collections类

java.util.Collections类提供了一些静态方法实现基于List容器的一些常用算法。

  • void sort(List)//排序
  • void shuffle(List)//随机排序
  • void reverse(List)//逆序排序
  • void fill(List, Object)//用一个特定对象重写整个容器
  • void copy(List dest, List src)//将src List容器拷贝到dest List容器
  • int binarySearch(List, object)//折半查找

final

不允许修改的变量；

基础类型占一个内存

引用类型占两个内存

引用类型全为指针；对象是new出来的，放在堆内存。一小块内存指向一大块。

构造方法没有返回值

方法执行完毕，栈内存中的局部变量销毁

static声明的成员变量：
可通过类名直接访问， 静态方法不针对某个对象调用，所以不能访问非静态成员

子类的构造过程中必须调用父类构造方法

子类中使用super(argument_list)调用父类构造方法；
只用this(argument_list)调用本类的构造方法；
如果调用了super，必须写在子类构造方法内的第一行。
如果子类的构造方法没有显示调用父类构造方法，则系统默认调用父类的无参构造方法，而若父类中没有无参构造方法，则编译出错。

new 出来的对象永远不“==”
.equals()重写后可以用于比较；
new String() equals 相等是因为重写了该方法；

没有import即可使用的类一定在java.lang中

hash 编码用于存储对象在堆内存中的地址

对象转型：
一个基类的引用不可以访问其子类对象新增的成员；
可以使用变量 instanceof 类名来判断引用类型变量所指向的对象是否属于该类或是该类的子类，子类的对象可以作为基类的对象来使用 称作向上转型，反之称为向下转型

多态：
在执行期间 判断所引用的实际类型，根据其实际的类型（即实际new 的class类型，或）调用相应的方法。（方法重写时）。

存在的三个条件

  1. 要有继承。
  2. 要有方法重写。
  3. 父类引用类型指向子类对象。

抽象方法 ： 用于多态的方法。 abstract
抽象类不能实例化对象， 抽象方法只需声明， 不需实现。
抽象方法只用于被重写。

final 的变量：不能被改变；
final 的方法：不能被重写；
final 的类：不能被继承。

接口

一种特殊的抽象类。只包含常量和方法的定义，没有变量和方法的实现。

  1. 接口可以多重实现
  2. 接口中的成员变量属性为public static final；
  3. 接口中只能定义抽象方法，且方法只能为public；
  4. 接口可以继承其他接口，并添加新属性和抽象方法。
